# Phase 2: Domain Capture Checklist

## Pre-Work
- [ ] Discovery Doc reviewed
- [ ] Data access confirmed
- [ ] Domain expert interviews scheduled
- [ ] Existing documentation gathered

## Terminology Capture
- [ ] Core terms listed
- [ ] Client definitions documented (in their words)
- [ ] Common confusions identified
- [ ] Examples provided for each term
- [ ] Status values mapped
- [ ] Acronyms defined

## Business Rules Extraction
- [ ] Rules organized by domain area
- [ ] Each rule clearly stated
- [ ] Business reason documented (why it exists)
- [ ] Examples provided
- [ ] Anti-patterns identified for each rule

## Data Semantics
- [ ] All data sources/tables documented
- [ ] Field-by-field definitions created
- [ ] Data types confirmed
- [ ] Valid values documented
- [ ] Gotchas/quirks noted
- [ ] Relationships mapped

## Known Entities
- [ ] Entity types identified (locations, statuses, categories, etc.)
- [ ] IDs/codes catalogued with names
- [ ] Notes added for special cases
- [ ] Client validated completeness

## Edge Cases
- [ ] Edge cases identified through interviews
- [ ] Each scenario documented
- [ ] Why it's tricky explained
- [ ] Correct handling specified
- [ ] Examples provided

## Anti-Patterns
- [ ] Common mistakes identified
- [ ] Wrong approaches documented
- [ ] Why they're wrong explained
- [ ] Correct approaches specified
- [ ] Detection methods noted

## Calculations
- [ ] Key calculations identified
- [ ] Formulas documented exactly
- [ ] Inputs specified
- [ ] Gotchas noted
- [ ] Examples with actual numbers provided

## Query Templates
- [ ] Common query types identified
- [ ] SQL/query templates written
- [ ] When to use documented
- [ ] Gotchas noted

## Validation
- [ ] Terminology validated with client
- [ ] Business rules validated with client
- [ ] Data semantics validated with client
- [ ] Edge cases validated with client
- [ ] Full document reviewed by domain expert

## Deliverable
- [ ] DOMAIN_SPEC.md created using template
- [ ] All sections completed
- [ ] Client sign-off obtained
- [ ] Version control established

## Phase Exit Criteria
**Phase 2 is complete when ALL of the following are true:**
- [ ] Domain Spec is finished
- [ ] All terms have client-validated definitions
- [ ] All business rules are documented with examples
- [ ] All data fields have definitions and gotchas
- [ ] Known entities are catalogued
- [ ] Edge cases are documented with handling
- [ ] Anti-patterns are documented
- [ ] Client has reviewed and signed off
- [ ] Domain Spec is ready to be encoded into Cortex
