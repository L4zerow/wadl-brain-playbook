# Phase 5: Iterate Checklist

> This phase is ongoing. Use this checklist for each iteration cycle.

## Iteration Setup
- [ ] Iteration number assigned
- [ ] Date range defined
- [ ] Focus area identified
- [ ] Success criteria for this iteration defined

## Feedback Collection
- [ ] User feedback gathered (surveys, interviews, support tickets)
- [ ] Usage metrics reviewed
- [ ] Error logs analyzed
- [ ] Common query patterns identified
- [ ] Failed queries categorized

## Issue Triage
- [ ] All issues logged
- [ ] Severity assigned (High/Med/Low)
- [ ] Root cause identified
- [ ] Priority set for this iteration

## Cortex Updates
- [ ] New edge cases identified
- [ ] New anti-patterns discovered
- [ ] Business rules needing clarification found
- [ ] Updates drafted
- [ ] Updates tested with sample queries
- [ ] Updates deployed

## Domain Spec Updates
- [ ] New learnings documented
- [ ] Terminology updates made
- [ ] Business rule clarifications added
- [ ] Edge cases added
- [ ] Domain Spec version updated

## Feature Implementation (if applicable)
- [ ] Feature requirements clear
- [ ] Design completed
- [ ] Implementation done
- [ ] Testing completed
- [ ] Deployed

## Bug Fixes (if applicable)
- [ ] Bug reproduced
- [ ] Root cause identified
- [ ] Fix implemented
- [ ] Fix tested
- [ ] Fix deployed
- [ ] User notified

## Testing
- [ ] Changes tested in staging
- [ ] Regression tests passed
- [ ] User validation obtained
- [ ] Production smoke tests passed

## Documentation Updates
- [ ] Iteration Log updated
- [ ] Domain Spec updated (if changed)
- [ ] Cortex version documented
- [ ] Known issues updated

## Communication
- [ ] Stakeholders notified of changes
- [ ] User-facing changes communicated
- [ ] Support documentation updated

## Metrics Review
- [ ] Query success rate checked
- [ ] Response time checked
- [ ] Error rate checked
- [ ] User satisfaction checked (if measured)

## Iteration Retrospective
- [ ] What went well documented
- [ ] What didn't go well documented
- [ ] Learnings captured
- [ ] Process improvements identified

## Next Iteration Planning
- [ ] Backlog reviewed
- [ ] Next iteration focus identified
- [ ] Dependencies identified
- [ ] Resources confirmed

---

## Weekly Checklist (Ongoing)
- [ ] Error logs reviewed
- [ ] User feedback checked
- [ ] Health metrics verified
- [ ] Any urgent issues addressed

## Monthly Checklist (Ongoing)
- [ ] Monthly review completed
- [ ] Metrics summarized
- [ ] Key wins documented
- [ ] Challenges documented
- [ ] Next month priorities set
- [ ] Client status update sent

## Quarterly Checklist (Ongoing)
- [ ] Quarterly business review completed
- [ ] ROI assessment done
- [ ] Strategic roadmap reviewed
- [ ] Contract/engagement reviewed
- [ ] Major improvements planned

---

## Iteration Exit Criteria
**An iteration is complete when:**
- [ ] All planned changes are deployed
- [ ] Testing is complete
- [ ] Documentation is updated
- [ ] Stakeholders are notified
- [ ] Iteration Log is updated
- [ ] Next iteration is planned
